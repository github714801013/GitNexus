# Zoekt 400 Fix Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 修复 MCP 代理调用 Zoekt `/api/search` 时只记录 `HTTP 400` 的服务端异常，并让 Docker 中 Zoekt JSON API 按官方要求可用。

**Architecture:** 保持 GitNexus MCP 工具入口不变，仅调整 Zoekt 部署启动参数和客户端错误诊断。`zoekt-webserver` 显式使用 `-rpc` 暴露 JSON API，并用 `-index /data/index` 读取 compose 挂载的索引目录。

**Tech Stack:** TypeScript, Vitest, Docker Compose, Zoekt JSON API

---

## Scope

- 修改 `mcp_proxy_docker/docker-compose.yml`：为 `zoekt-webserver` 增加显式 `command`。
- 修改 `gitnexus/src/core/search/zoekt-client.ts`：HTTP 非 2xx 时读取响应正文，日志能显示 Zoekt 返回的具体错误。
- 修改 `gitnexus/test/unit/zoekt-client.test.ts`：覆盖 HTTP 400 响应正文被记录的路径。
